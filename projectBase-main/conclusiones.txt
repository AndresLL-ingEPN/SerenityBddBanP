================================================================================
  RESUMEN EJECUTIVO
================================================================================

El proyecto automatiza el flujo de compra en Swaglabs utilizando Serenity BDD
con patron Screenplay sobre Microsoft Edge. La estructura de capas es correcta
y el patron Actor-Task-UI esta bien aplicado. Sin embargo, se identificaron
hallazgos criticos que impedian la ejecucion, problemas en la estrategia de
localizadores, y ausencia de cobertura en escenarios negativos. Todos los
hallazgos de codigo fueron corregidos. Los de estrategia de pruebas quedan
documentados como recomendaciones de mejora.


================================================================================
  HALLAZGOS DE AUTOMATIZACION - ESTRATEGIA DE LOCALIZADORES
================================================================================

[LA-01] USO DE XPATH CUANDO EL ELEMENTO TIENE ID DISPONIBLE
-------------------------------------------------------------
  Archivos  : PageCar.java (6 localizadores), PageProducts.java (2 localizadores)
  Problema  : La mayoria de localizadores usan XPath con el atributo @id en lugar
              de usar directamente el selector CSS por ID, que es la estrategia
              recomendada y la mas performante:

              Localizadores actuales (XPath):
                //button[@id='checkout']          PageCar.java
                //input[@id='first-name']         PageCar.java
                //input[@id='last-name']          PageCar.java
                //input[@id='postal-code']        PageCar.java
                //input[@id='continue']           PageCar.java
                //button[@id='finish']            PageCar.java
                //button[@id='add-to-cart']       PageProducts.java
                //button[@id='back-to-products']  PageProducts.java

              Como deberian ser (CSS ID):
                #checkout                         PageCar.java
                #first-name                       PageCar.java
                #last-name                        PageCar.java
                #postal-code                      PageCar.java
                #continue                         PageCar.java
                #finish                           PageCar.java
                #add-to-cart                      PageProducts.java
                #back-to-products                 PageProducts.java

  Contraste : PageMain.java SI usa correctamente selectores CSS por ID
              (#user-name, #password, #login-button). El mismo criterio
              debia aplicarse en el resto de Page Objects.

  Por que importa:
    - El selector por ID (#id) es el mas rapido en el DOM: O(1) en la mayoria
      de motores de busqueda del navegador.
    - XPath requiere recorrer el arbol DOM y es entre 3x y 10x mas lento.
    - Los selectores CSS por ID son menos fragiles ante cambios de estructura HTML.
    - Es la primera recomendacion de cualquier guia de automatizacion web
      (Selenium best practices, WebDriver W3C spec).

  Impacto   : MEDIO - No rompe la ejecucion pero afecta performance y
              mantenibilidad a medida que crece el suite.

[LA-02] LOCALIZADOR POR CLASE CSS FRAGIL
------------------------------------------
  Archivo   : PageProducts.java - BUTTON_CART
  Localizador actual: //a[@class='shopping_cart_link']
  Problema  : Localizar por clase CSS es fragil; si el equipo de desarrollo
              renombra o agrega clases, el selector se rompe. El elemento
              tiene un contenedor con ID 'shopping_cart_container' que
              seria mas estable: #shopping_cart_container a
  Impacto   : BAJO - Funciona hoy, riesgo de ruptura ante refactoring de UI.

[LA-03] LOCALIZADOR POR TEXTO VISIBLE
---------------------------------------
  Archivos  : PageProducts.java (SELECTED_PRODUCT), PageMain.java (LABEL_PRODUCT),
              PageCar.java (LABEL_SUCCESSFULL_PURCHASE)
  Problema  : Tres localizadores dependen del texto visible del elemento:
                //div[normalize-space()='{0}']
                //span[contains(text(), 'Products')]
                //h2[normalize-space()='Thank you for your order!']
              El texto es el atributo mas volatil de una UI: cambia con
              internacionalizacion, correcciones de redaccion o rebranding.
  Evaluacion: Para SELECTED_PRODUCT no hay alternativa clara ya que los
              elementos de producto no tienen ID propio. Para los otros dos,
              se podria usar atributos data-test o data-cy si el equipo de
              desarrollo los expone.
  Impacto   : BAJO-MEDIO segun la madurez del equipo de desarrollo.


================================================================================
  ASPECTOS POSITIVOS
================================================================================

[+] Patron Screenplay bien aplicado. La separacion Actor-Task-UI es correcta
    y facilita la reutilizacion de Tasks entre escenarios.

[+] LocalEdgeDriverEnhancer implementa la API BeforeAWebdriverScenario
    correcta para Serenity 4.x. La migracion desde la API deprecada
    WebDriverManager fue realizada correctamente.

[+] SELECTED_PRODUCT usa Target parametrizable con {0}, lo que permite
    reutilizar el mismo localizador para cualquier producto del catalogo
    sin duplicar codigo.

[+] ModelCredentials mapea el DataTable de Cucumber a un objeto de dominio,
    lo que desacopla la capa de datos del step definition.

[+] serenity.conf tiene configuracion multi-entorno, opciones de Edge
    correctamente declaradas y captura de pantalla activada por accion
    (FOR_EACH_ACTION), lo que maximiza la evidencia en el reporte HTML.

[+] FeatureOverwrite es un mecanismo propio para inyeccion de datos externos
    en features. Demuestra conocimiento avanzado del ciclo de vida de Cucumber.

[+] La infraestructura de logging con Logback esta correctamente configurada
    y el uso de @Slf4j via Lombok es consistente en todo el proyecto.
